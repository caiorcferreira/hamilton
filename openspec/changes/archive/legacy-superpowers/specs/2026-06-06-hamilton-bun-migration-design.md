# Hamilton Bun Migration тАФ Design

Date: 2026-06-06

## Summary

Port Hamilton from npm/Node.js to bun. Replace `better-sqlite3` (C++ native addon) with bun's built-in `bun:sqlite`, switch package manager to bun, add `install-local` and `purge` scripts for easy local installation and cleanup.

## Section 1: Package manager & deps

### package.json changes

- `better-sqlite3` removed тАФ replaced by bun's built-in `bun:sqlite`
- `@types/better-sqlite3` removed тАФ `bun:sqlite` ships its own types
- All version ranges pinned (`^` тЖТ exact) to match existing convention for `pi-agent-core`, `effect`, `yaml`
- New scripts:
  - `"install-local": "bun run build && bun link"` тАФ builds + makes `hamilton` globally available
  - `"purge": "bun unlink hamilton; rm -rf ~/.hamilton/"` тАФ removes binary + all Hamilton data

### Lockfile

- `package-lock.json` deleted
- `bun.lockb` generated by `bun install`

### .gitignore

Add `bun.lockb`.

### tsconfig.json

No changes тАФ `tsc` compiles as before, `bun` runs the output. `module: "Node16"` stays.

## Section 2: better-sqlite3 тЖТ bun:sqlite migration

7 files import `better-sqlite3`. The bun:sqlite API is near-identical:

| current (better-sqlite3) | bun:sqlite |
|---------------------------|------------|
| `import Database from "better-sqlite3"` | `import { Database } from "bun:sqlite"` |
| `new Database(path)` | `new Database(path)` |
| `db.pragma("journal_mode = WAL")` | `db.run("PRAGMA journal_mode = WAL")` |
| `db.prepare(sql).run(params)` | same |
| `db.prepare(sql).get(params)` | same |
| `db.prepare(sql).all(params)` | same |
| `db.exec(sql)` | `db.run(sql)` (or `db.exec(sql)` for multi-statement) |
| `db.close()` | same |

### Type changes

All `Database.Database` type annotations become `Database` тАФ bun:sqlite exports the class directly, not as a namespace.

### Files changed

| File | Changes |
|------|---------|
| `src/db/schema.ts` | Import + pragma line 39 |
| `src/db/queries.ts` | Import + `Database.Database` тЖТ `Database` in 3 function signatures |
| `src/workflow/state.ts` | Import + pragma line 39 + `openDb()` return type |
| `src/workflow/run-state-machine.ts` | Import line only |
| `tests/db/schema.test.ts` | Import + `Database.Database` тЖТ `Database` + temp db helper |
| `tests/db/queries.test.ts` | Import + `Database.Database` тЖТ `Database` |
| `tests/cli/status.test.ts` | May use `openDb` indirectly тАФ verify |

### Test compatibility

The `tempDb()` helper in `tests/db/schema.test.ts` uses `new Database(dbPath)` + stores metadata via `(db as any)`. All of this works identically with bun:sqlite.

vitest stays тАФ no changes to test framework or assertions.

## Section 3: install-local and purge scripts

### install-local

```
bun run build && bun link
```

- `bun run build` executes `tsc -p tsconfig.json` via bun
- `bun link` creates a global symlink referencing the `"bin"` field (`dist/cli/main.js`), making `hamilton` available anywhere

### purge

```
bun unlink hamilton && rm -rf ~/.hamilton/
```

- Removes the global `hamilton` binary symlink
- Removes all Hamilton data: agents, workflows, DB, runs, configs, step outputs, logs
- Project directory intact for reinstall

### Usage flow

```bash
git clone <repo> && cd hamilton
bun install                    # install deps, generates bun.lockb
bun run install-local          # build + globally link hamilton
hamilton init                  # bootstrap ~/.hamilton/
# ... use it ...
bun run purge                  # remove everything, start fresh
```

## Implementation order

1. Delete `package-lock.json`, add `bun.lockb` to `.gitignore`
2. Migrate `better-sqlite3` тЖТ `bun:sqlite` in all 7 files
3. Update `package.json`: remove deps, pin versions, add scripts
4. Run `bun install` to generate `bun.lockb`
5. Run `bun run build && vitest run` тАФ verify all 133 tests pass
6. Test `bun run install-local && hamilton init` end-to-end
