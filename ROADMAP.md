  - Planner LLM yields tool calls that explore the codebase (bash, file search, grep) but never produces the expected `STATUS: done` + `STORIES_JSON` output, so `parseAgentOutput` (`src/agent/activity.ts`) keeps failing
  - Every turn logs 0 tokens (`tokens_in: 0, tokens_out: 0`) via `subscribePiEvents` in `pi-executor.ts:145`, despite `llm_delta` events streaming — potential Pi SDK bug in token accounting, or the response is being treated as pure tool-call output with no text content (triggers `PiExecutionError("Assistant response had no text content")` at `pi-executor.ts:174`)
  - After the 4th retry the retry schedule should exhaust and `executeWithPi` should fail, transitioning the step to `failed` in `runner.ts:161-164` and the run to `failed` — but all 3 runs are still `running` in the DB, meaning the Effect pipeline is either stuck in a retry loop or failed silently without calling `ctx.transitionStep(stepId, "fail")`
  - A 4th run (`feature-dev-060fef5c`) produced a run-id visible to the user but was never persisted to `hamilton.db` or any run directory — likely generated by the `catchAll` fallback in `run.ts:48-57` which synthesizes a failed `WorkflowResult` with a fresh UUID via `buildRunId()` without ever inserting a row; this path masks the real error
  - **Root cause suspects:** (a) `parseAgentOutput` rejecting planner output that has tool calls but no final `STATUS: done` text block, (b) retry schedule not exhausting correctly (the `Effect.retry` + `Effect.timeout` interaction at `runner.ts:140-156` may not propagate failures through the chain), (c) the `catchAll` in `run.ts` discards the original error and creates a phantom run-id, making debugging harder
  - **Areas to fix:** add logging of the actual failure reason before the `catchAll` in `run.ts`; verify `Effect.timeout` + `Effect.retry` composition propagates errors correctly; ensure `ctx.transitionStep(stepId, "fail")` and `ctx.fail(workflowStatus)` are always called on exhaustion; add a sentinel to detect planner output that only contains tool calls and no final text answer
